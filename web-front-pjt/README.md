- **목적:**

## 과제별 기능 설명

### 1. 식단 기록 및 관리

- **식단 기록**
  - 날짜, 식사 종류(아침/점심/저녁/간식), 음식 이름, 영양소(칼로리/탄수화물/단백질/지방) 입력
  - 음식 이름 자동완성(내장 foodDB.csv 기반)
  - 기록된 식단은 날짜별로 저장됨

- **식단 수정/삭제**
  - 기록된 식단 항목을 수정/삭제 가능
  - 날짜, 음식명, 영양소 등 모든 항목 수정 가능
  - 수정 시 분석 및 통계에 즉시 반영

- **식단 목록**
  - 전체 식단 기록을 날짜순으로 확인 가능
  - 각 기록별 상세 정보 및 수정/삭제 버튼 제공

---

### 2. 식단 분석 및 대시보드

- **홈 대시보드**
  - 오늘의 식단 요약(총 섭취 칼로리, 탄수화물, 단백질, 지방)
  - 영양 목표(사용자별 목표값)
  - 오늘의 식단 상세(식사별 기록 리스트)
  - 주간 통계(최근 7일간 섭취량 표)
  - 주간 영양소 섭취 현황(Chart.js 그래프)

- **분석 탭**
  - 오늘의 섭취량 막대그래프
  - 주간 영양소 비율 파이차트
  - 일별 영양소 트렌드 라인차트
  - 주간 섭취량 비교 바차트

- **분석 반영**
  - 식단 기록/수정/삭제 시, 모든 분석 결과가 실시간 반영됨
  - 날짜별 기록이 정확히 분석에 포함됨

---

### 3. 프로필 관리

- **프로필 정보**
  - 사용자 아이디, 키, 몸무게, 질환 등 입력 및 수정 가능
  - 영양 목표(칼로리, 탄수화물, 단백질, 지방) 설정 가능(확장 가능)
- **계정 관리**
  - 회원 탈퇴, 계정 비활성화 기능 제공

---

### 4. 소셜(팔로우) 기능

- **팔로우/팔로워**
  - 다른 사용자를 팔로우/언팔로우 가능
  - 팔로잉/팔로워 목록 확인
- **소셜 탭**
  - 친구의 식단 기록, 프로필 등 확인(확장 가능)

## 설치 및 실행

1. **프로젝트 클론**
   ```
   git clone https://lab.ssafy.com/thj71300187/webfront_seoul_16_binj.git
   cd webfront_seoul_16_binj
   ```

2. **foodDB.csv 파일 확인**
   - 프로젝트 루트에 `foodDB.csv` 파일이 있어야 합니다.

3. **로컬 서버 실행**
   - VSCode의 Live Server 확장 또는 아래 명령어로 실행:
     ```
     python -m http.server
     ```
   - 브라우저에서 `http://localhost:8000` 접속

4. **회원가입 후 로그인하여 사용**
